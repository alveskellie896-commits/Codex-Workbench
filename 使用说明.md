# Codex Workbench 手机使用说明

这份说明面向日常使用，不要求你懂环境变量或底层实现。把它加到 iPhone 主屏幕后，日常路径应该是：打开图标或稳定网址 → 自动可信登录/恢复 → 进入项目或最近会话。

## 1. 这是什么

Codex Workbench 是一个 Windows 笔记本 + iPhone 的远程 Codex 工具：

- Windows 笔记本运行本地桥接服务，连接 Codex Desktop / Codex CLI / Codex app-server，读取本机 `~/.codex` 会话。
- iPhone 打开 PWA 网页，可以看项目、会话、消息、运行状态、队列、附件、Git、Subagents 和诊断。
- Codex 仍在你的电脑上运行；手机只是安全遥控器。

电脑必须保持开机、联网、不休眠，并且 Workbench 服务在线。电脑关机、睡眠、断网时，手机打不开是正常的；队列不会故意丢，但要等恢复后同步。

## 2. Windows 端怎么启动

普通本地启动：

```powershell
npm install
npm run build
npm start
```

跨网络手机访问推荐：

```powershell
npm run start:public
```

然后打开项目根目录的 `current-phone-link.txt`，看 `Phone:` 那一行。

`current-phone-link.txt` 会告诉你：

- `Phone`：iPhone 要打开的网址。
- `Computer`：电脑本机网址。
- `TunnelType`：Tailscale / Cloudflare / localhost.run / custom / recovering。
- `FailureReason`：如果隧道坏了，写明原因。

换 API key 通常不会改变手机网址。网址是否变化取决于隧道类型：Tailscale/Funnel 通常稳定；Cloudflare/localhost.run 临时链接可能变化。

## 3. 第一次设置密码

第一次打开网页时，先设置访问密码。这个密码保护你的电脑端 Workbench，不是 OpenAI API key。

忘记密码时，需要回到电脑端重新配置；不要把 `.env` 或 token 发给别人。

## 4. iPhone 第一次配对

1. 在电脑端打开 Workbench 并登录。
2. 点 `Advanced` → `Pairing` → `Create pairing code`。
3. iPhone 扫标准二维码，或打开配对链接，或手动输入短码。
4. 配对成功后，iPhone 会保存可信设备 token，并回到主界面。
5. 以后打开 Home Screen 图标或稳定 URL，优先使用可信登录。

配对码会过期，只能用一次，用过立即失效。失败时回电脑端重新生成即可。

## 5. 免密码进入和设备管理

如果手机看到登录页：

1. 点 `Trusted login`。
2. 如果信任仍有效，会自动进入。
3. 如果失败，用密码登录或重新扫码配对。

管理设备：

- `Advanced` → `Pairing` 可以看所有设备。
- 可重命名设备。
- 可撤销设备。
- 可在本机点 `Forget this browser` 忘记本浏览器 token。

撤销后，该手机 HTTP token 和 WebSocket 都会失效，下次需要密码或重新配对。

## 6. 怎么像 App 一样用

建议在 iPhone Safari 中点分享按钮 → 添加到主屏幕。PWA 已配置：

- 主屏图标和名称。
- 独立显示模式。
- safe-area 适配。
- 底部 composer 固定。
- 页面回前台自动同步。
- 新前端版本自动刷新旧缓存。

如果页面看起来旧或异常，先刷新；仍不行时打开诊断并清理 PWA 缓存。

## 7. 看项目、会话和长聊天

打开后第一屏是项目/会话/状态。

- 项目和会话按最近活跃排序。
- 可搜索。
- 会话有运行中、完成、失败、排队、未读/需要处理等状态提示。
- 打开会话默认看到最新消息。
- 向上滚动加载历史。
- 你正在看历史时，新流式回复不会强行把你拉到底部；会显示跳到底部按钮。

## 8. 发消息、停止和继续追问

在底部输入框发消息。

状态会按阶段显示：

- 本地已排队。
- 读取附件。
- 上传附件。
- 已发送到 Windows。
- Codex 已接收。
- Codex 处理中。
- 已回复。
- 失败可重试。
- 已取消。

当前任务运行中继续发消息，会进入 follow-up 队列，不会无提示消失。任务结束后会自动发送下一条；如果无法确认结束，会给重试/取消/查看原因。

## 9. 编辑、取消、重排队列

进入 `Advanced` → `Queue`：

- `Edit` 修改排队消息。
- `Up` / `Down` 调整顺序。
- `Cancel` 取消。
- 每条队列会显示目标线程、创建时间和状态。

自动重试有上限，不会无限 retry。刷新页面后，旧的 sending 状态会进入恢复检查；太旧的会停止自动重试并提示你手动处理。

## 10. Plan / Reasoning / Access 怎么用

进入 `Advanced` → `Run`：

- `Model`：选择模型。
- `Reasoning`：low / medium / high / xhigh。
- `Access mode`：
  - `Read-only`：尽量只读，不写文件。
  - `Ask before risky actions`：可编辑项目，但风险操作仍需确认。
  - `Full access`：只用于你完全信任的本地项目，保存前会二次确认。
- `Plan mode`：没有稳定原生 API 时使用 prompt fallback，会明确告诉 Codex 先规划。

注意：desktop 模式无法真实传入所有 reasoning/access 参数，界面会标注降级；CLI/app-server 模式能传的才会真实传。

## 11. Subagents 怎么用

进入 `Advanced` → `Agents`：

- 已存在的 native subagents 会按父子关系显示。
- 可以从父线程跳到子线程，也能回到父线程。
- 新建时选择 worker / explorer / reviewer / tester / docs 等角色模板。
- 当前没有稳定 native create API 时，会预览并发送 `/subagents` 命令；它是命令 fallback，不假装真的启动内部 agent。

## 12. Git 面板怎么安全用

进入 `Advanced` → `Git`。

默认只读信息：

- 当前分支。
- 是否 Git 仓库。
- 工作区是否干净。
- changed files。
- diff shortstat。
- 最近 commits。
- remote/upstream 状态。

写操作需要输入确认文字：

```text
confirm:commit
confirm:pull
confirm:push
confirm:checkout
confirm:create-branch
confirm:stash
```

默认禁止 reset、clean、force push、delete branch、rebase、危险 checkout。遇到冲突、无 upstream、未配置用户名邮箱、无权限或网络失败时，会显示人话提示。

## 13. 文件、图片、PDF、docx、相机、语音

底部 composer 和 `Advanced` → `Files` 都支持附件。

- 图片：显示图片卡片/预览。
- PDF：显示 PDF 卡片、大小和类型。
- docx：显示 Word 卡片、大小和类型。
- 相机：点相机入口；是否直接拍照取决于浏览器。
- 语音：浏览器支持 MediaRecorder 时可录音并作为附件；不支持时，用 iPhone 语音备忘录录好后作为文件上传。
- 上传失败可重试或移除。
- 附件只属于当前线程，不会串到其他会话。
- 刷新后浏览器可能丢失 File 对象，此时需要重新选择文件。

## 14. 通知和完成感知

Workbench 支持：

- 会话列表状态徽章。
- 站内通知中心。
- 可选浏览器通知。
- 页面标题/前台恢复同步。
- 电脑离线、隧道恢复、token 过期、队列卡住等提示。

它不伪装 APNs。iOS PWA Web Push 需要 Home Screen 安装、权限、订阅和服务端推送配置；当前默认用站内通知和前台恢复。

## 15. 诊断怎么用

点顶部诊断按钮，可以看到：

- Windows bridge 状态。
- 手机 URL 和隧道类型。
- WebSocket 是否在线。
- Codex CLI / Desktop bridge / app-server 状态。
- token 和可信设备数量。
- PWA 版本。
- 数据库、上传、服务检查。

红色是必须处理，黄色是有风险。诊断里的 URL 可以复制。

## 16. 开机自动启动

安装：

```powershell
.\scripts\install-autostart.ps1
```

取消：

```powershell
.\scripts\uninstall-autostart.ps1
```

日志：

- `autostart.log`：开机启动、重复启动保护、重启记录。
- `public-link.log`：隧道和服务输出。
- `current-phone-link.txt`：当前手机网址。

脚本只在项目 scripts 范围内工作，不修改系统关键安全策略。

## 17. 仍然不是 Remodex 原生 iOS 的部分

已经有现实替代：

- 可信设备登录替代原生 Keychain 体验。
- 站内通知/浏览器通知替代 APNs。
- Tailscale/Funnel/临时隧道替代独立 relay。
- VisualViewport + safe-area 替代 SwiftUI keyboard layout。
- Git 安全面板替代 Remodex 更完整 workspace checkpoint。

仍无法等同：

- APNs 级后台推送。
- 原生 iOS 后台常驻。
- 完整 E2E relay 密钥协议。
- 原生 iOS 文件分享扩展和系统级语音转写。
- Windows 环境下 Xcode 真机构建验证。

要做真正原生 iOS App，需要 Mac、Xcode、开发者证书、真机测试、推送配置，并复用当前 Windows bridge API。

## 18. 出问题先看这里

- 手机打不开：看 `current-phone-link.txt` 是否更新，电脑是否开机联网，隧道是否 recovering。
- 电脑睡眠/关机：手机无法连接，等电脑恢复后刷新或重新打开 PWA。
- 网址变了：打开 `current-phone-link.txt` 使用新的 `Phone` 地址。
- API key 换了：通常不影响手机网址，但 Codex 本身可能要重新登录。
- 发消息一直转：看队列状态；失败后点重试/取消/查看原因。
- 附件丢失：重新选择文件。
- token 过期：可信登录或密码登录。
- 设备被撤销：重新输入密码或重新配对。
