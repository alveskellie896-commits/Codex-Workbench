## 总体目标
把当前 Codex-Workbench 从“手机网页遥控器”升级为一个尽量对齐 Remodex 的完整产品，但目标平台不是 Remodex 的 Mac + iPhone，而是 **Windows 笔记本 + iPhone**。

最终用户体验目标：
- Windows 笔记本运行一个可靠的本地桥接服务，负责连接 Codex Desktop / Codex CLI / Codex app-server、读取本机 `~/.codex` 会话、执行安全的 Git/文件/诊断操作、维持公网或私网可达链接。
- iPhone 端提供接近原生 App 的 PWA/网页体验；如果仓库内 iOS SwiftUI 外壳可合理维护，也补齐可复用的 iOS 源码，但不能要求用户必须有 Mac/Xcode 才能用。
- iPhone 和 Windows 可以不在同一个 Wi-Fi，只要两边联网且 Windows 服务在线，就能通过稳定 URL 或可信 relay/Tailscale/Funnel 方案访问。
- 第一次配对尽量像 Remodex：电脑端生成标准二维码/短码，iPhone 扫码或打开链接后信任这台 Windows 笔记本；以后优先免密码进入，密码作为恢复入口。
- 手机端可以查看项目、会话、运行状态、聊天历史、操作记录摘要、子任务、队列、附件、Git 状态，并能发送消息、传文件、排队追问、停止任务、执行安全确认后的 Git 操作。
- 长聊天记录打开要快，流式回复不能让页面乱跳，iPhone Safari 键盘不能把输入框挤没。
- 错误、断网、token 过期、服务重启、隧道失效、数据库繁忙、WebSocket 断开，都要尽量自动恢复；无法恢复时必须给用户傻瓜式提示。
- 最终交付必须可运行、可测试、可说明。不要只写报告，不要只做按钮，不要假装浏览器能力等同原生 iOS。

本任务要求“一步到位”，但不是允许胡乱堆功能。执行 AI 必须用工程判断把 Remodex 能力分成：
- **Windows+iPhone 可真正实现**：必须真实实现。
- **可用网页/PWA 替代**：必须做成清楚、不误导、可长期使用的替代方案。
- **必须原生 iOS / APNs / 独立 relay / macOS 才能实现**：不得伪装完成；必须写清楚边界，并尽量提供 Windows+iPhone 的现实替代路径。

### Remodex 近乎原生体验硬性目标
本任务最终不能只交付“网页功能增强版”。执行 AI 必须把最终产品验收到“普通用户把它加到 iPhone 主屏幕后，日常使用感接近 Remodex 原生 iOS App”的水平。

近乎原生体验在本任务中定义为：
- iPhone 端主入口必须像 App，而不是像调试网页：支持 PWA 安装提示、主屏图标、启动名称、独立显示模式、safe-area、稳定底部 composer、无明显浏览器感布局。
- 用户日常路径必须少于三步：打开 iPhone 图标或稳定 URL → 自动可信登录/恢复 → 进入最近会话或项目列表。
- 首次配对必须接近 Remodex：Windows 端显示标准二维码和短码；iPhone 扫码/打开链接后完成可信设备绑定；以后优先免密码进入。
- 会话列表、聊天页、运行状态、队列、附件、Git、Subagents、诊断必须整合为一个连贯的手机产品，不能把高级能力丢在割裂的“外挂面板”里。
- iPhone Safari/PWA 键盘体验必须作为一级验收项：输入框可见、底部不大面积留白、不乱跳、不遮挡、不因为中文输入法候选栏而失控。
- 长线程体验必须接近原生：秒开最近消息、向上加载历史、虚拟滚动、流式回复不打断历史阅读。
- 所有状态必须可感知：运行中、排队中、已送达 Windows、Codex 处理中、已回复、失败、需要处理、电脑离线、隧道恢复中。
- 断线/恢复必须接近原生 App：页面回前台后自动同步，WebSocket 自动重连，token 可刷新，旧缓存能自修复，旧 spinner 不残留。
- 文件体验必须像手机 App：图片预览、PDF/docx 文件卡片、相机入口、语音入口或明确降级、上传进度、失败重试、当前线程隔离。
- Remodex 里容易形成“原生感”的细节不能漏：Sidebar/会话分组、运行状态徽章、队列草稿、命令/文件/技能补全、上下文用量、账号/速率状态、Git 安全操作、工作区变更预览/回滚、连接恢复卡片、完成提醒、可信重连、断线 catch-up。
- 如果 PWA 无法达到上述体验阈值，执行 AI 必须把 `ios\CodexWorkbench` 作为一级路线评估和增强目标，而不是一句“可选”。在 Windows 环境不能 Xcode 构建时，也必须至少补齐可复用 API、Swift 源码结构、Host 配置、登录/配对、消息列表、composer、附件和状态视图，并明确需要 Mac/Xcode 真机验证的剩余步骤。
- 最终报告必须给出“Remodex 对标表”：Remodex 能力、Windows+iPhone 实现方式、是否真实实现、是否 PWA 替代、剩余差距、用户实际影响。
- 如果某项无法达到 Remodex 原生级别，不能只写“受限无法实现”；必须给出当前 Windows+iPhone 下最接近的替代方案、用户怎么用、风险是什么、后续要变成原生 iOS 还需要什么。

### 原生体验评分闸门
执行 AI 必须建立并填写一个 100 分的 Remodex 近乎原生体验评分表。最终总分低于 85 分时，不得声称“近乎一样的原生体验已完成”；必须继续修复可修项目，或明确标记任务未完全完成。

评分维度必须至少包括：
- 首次配对与可信重连，10 分：标准二维码、短码兜底、一次性过期、可信设备、免密码恢复、撤销断连。
- iPhone 主屏 App 感，10 分：PWA manifest、图标、启动名称、独立显示、安全区域、返回/导航、无明显浏览器调试感。
- 会话列表与项目导航，10 分：搜索、置顶、最近活跃、状态徽章、父子任务、未读/完成/失败清楚。
- 聊天页与长记录，10 分：默认最新消息、历史增量加载、虚拟滚动、流式回复不打断阅读、操作记录折叠。
- composer 与 iOS 键盘，15 分：中文输入法、键盘弹出、safe-area、附件、发送/停止、Plan/运行控制、底部不消失、不大黑边。
